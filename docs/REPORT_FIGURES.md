# Report Figure Manifest

The design report uses HTML and MATLAB generated figures to make the embedded
engineering structure readable under a short preliminary video limit.

| Figure | Purpose |
| --- | --- |
| Dual RDK X5 architecture | Shows the AI brain, embodied brain, MCU layer, fixed arm station, and evidence site. |
| Requirement and implementation workflow | Separates material prediction, perception, station execution, evidence return, and safety boundary. |
| AI brain data flow | Shows image, XRD, spectrum, and material verdict lines entering the local decision layer. |
| Hardware topology | Maps sensors, compute boards, MCU, actuators, and fixed stations to their responsibilities. |
| Lab-FSD shadow planner | Explains occupancy, candidate paths, risk, and safety gate without claiming chassis takeover. |
| Fixed station and F407 execution | Shows arm01 vision gate, fixed point bag transfer, and F407 safety fallback. |
| Completion matrix | Separates measured functions, shadow or fallback functions, and later completion items. |
| Submission boundary | Shows video, report, code package, and public repository boundaries. |

The HTML figures include small icon elements to match the visual density of the
reference report while keeping all text and claims defensible.
